Problem Definations

Hardware token and alphanumeric password are the common user authentication for online banking. Using alphanumeric password with high password entropy for stronger and secure password creates burden to human memory for recalling complex password and create inconvient experiences for user to reset password. Many research has proved that using an image as a password has shows the potential of human memory to recall image password result with high retention and improve password security over complex alphanumeric password. Carrying around hardware token could create inconvient experience for user and unavailable hardware token for any reason could restrict the user to access. Common graphical password attacks such as shoulder-surfing, guessing, intersection has greatly impact the security of graphical password authentication.

Graphical password techniques implemented:

  • Recall based graphical authentication: The recall of password image is based on anchoring and relaying technique to aid the human memory, where any object exist in the image is select according to user selection and finally user will relay the meaning of the selected image based on user experiences of that image/object to be converted to text password using passpharases technique for high entropy password
  • Recognition based graphical authentication: Randomly shuffle two user selected image with decoy image within the 8 x 8 grid layout. Randomly display one of two user selected images with 16 decoy images
  • Dynamic graphical one-time password: Randomly generated graphical one time password , where the user must enter the OTP according to the location of user images based on recognition and user will need to enter the given OTP locations which is sent to the user registered email acocunt.
